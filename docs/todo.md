# やることリスト

## ゴール

GitHub Actions 上で agent-browser を使う際のキャッシュ戦略（CI高速化）を検証し、結果を `docs/report.md` にまとめる。

- キャッシュなし vs キャッシュあり でインストール時間を比較
- 単一ジョブでの計測（並列ジョブは対象外）
- 1回の実行で明確な差が出るかどうかを確認する

## 比較パターン

| # | パターン | 状態 |
|---|---|---|
| ① | キャッシュなし（ベースライン） | ✅ 完了 |
| ② | Chrome バイナリをキャッシュ | 未着手 |

> システム依存（`--with-deps` でインストールされる apt パッケージ等）のキャッシュは調査の過程で検討する

## ベースライン計測結果（参考）

全体: **43秒**

| ステップ | 所要時間 |
|---|---|
| agent-browser をインストール | 4秒 |
| Chrome とシステム依存をインストール | 33秒 ← ここが最大のボトルネック |
| ブラウザ操作（open → snapshot → close） | 2秒 |

## メモ

- Playwright の公式キャッシュ戦略はアーキテクチャの参考にする（実際には Playwright は使わない）
- agent-browser は Rust 製ネイティブバイナリ。Chrome は Chrome for Testing を別途ダウンロードする仕組み
- システム依存（apt パッケージ）はバイナリと違いキャッシュが難しい可能性あり
